得分分析

  • 擅长领域: 相图分析 (1.0)、力学性质 (1.0)、结构查询 (0.67)、数据库查询 (0.67) — 定性/描述性问题表现好
  • 薄弱领域: 符号计算 (0.0)、热力学计算 (0.0)、多尺度设计 (0.0)、降解分析 (0.0) — 需要精确数值或工具链的问题表现差
  • 根因: 工具调用预算耗尽后无法完成数值计算; SymPy/MP数据库工具在并发压力下调用成功率低

已知问题与分析

1. 后端事件循环阻塞 (严重)

当 4+ 个并发 LLM 请求同时到达后端时, 同步的 DeepSeek API 调用会阻塞 FastAPI 事件循环, 导致:

  • REST 端点超时 (curl exit code 28)
  • 新 WS 连接被拒绝 (ConnectionRefusedError)
  • 后端进程存活但 TCP 服务器不接受新连接

修复建议: 将 LLM 调用包装到 asyncio.to_thread() 或使用 run_in_executor(), 避免阻塞事件循环。这是架构级问题, 影响所有并发场景。

2. DeepSeek API 速率限制 (中等)

"Too many messages, slow down." 错误在 3 轨道 + ClawBench 同时运行时频繁出现。perovskite 轨道调研阶段因此超时, electrolyte 研究阶段连续触发 14 次。

缓解措施: 流水线已实现 graceful degradation (超时后继续下一阶段), 但需要客户端层面的请求队列化或 API 多 key 轮转。

3. ClawBench 得分偏低 (改进方向)

FinalScore 33.71, 9/11 案例未通过。主要问题:

  • 定量计算类 (带隙、生成焓、SymPy积分) 得分 0.0 — 工具调用未成功执行或结果未正确提取
  • 多尺度/研究设计类得分 0.0 — Agent 回复偏描述性, 缺少具体的可验证方案
  • 已有基准已扩展: MatSciBench (32 cases, 6 domains) + CSMBench (24 cases, 5 scales) — 更全面的评测覆盖
