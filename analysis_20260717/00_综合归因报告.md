## 〇、一句话回答

**不是模式切换（它从未发生）；工作流只答对了一半。跑分差是四层叠加：①分数本身有测量伪影（先别全信分数）→ ②执行闭环断裂（真实能力瓶颈）→ ③工作流/harness 配置压制（最大可修复杠杆）→ ④自进化与数学直觉两条回路从未接通（你的两个核心目标目前都是断线的）。**

## 二、真实能力瓶颈：执行闭环断裂（第一大直接失分项）

跨 benchmark 一致的行为签名：**「只描述不执行，且不知道（或不检查）自己没做出来」**。

1. **工具链摩擦烧光预算 → 占位交差**（RCB 三任务 100% 命中）：
   - M_000：bash_tool per-tool 50 次预算在数据加载试错中耗尽 → 报告交付 "Expected" 占位表 → judge 0 分；
   - M_003：Rust sandbox 对 RDKit+sklearn GP **静默崩溃**返回 "Unknown error"（回退只捕获异常、不捕获静默失败）→ 核心方法降级；
   - M_001：熔断器 circuit_open 自锁 → 弃 GNN 改 GP。
2. **「预算假耗尽」是占位报告的直接扳机**：预算其实按 chat 轮次自动重置（`streaming.py:1012-1015`），
   但错误消息无重置语义、LLM 不可见 → agent 误判「死刑」→ 写报告交差（`adapter.py:721-723`）。
3. **完成判据与 rubric 脱节**：`_is_done` 只查 3 个 glob + 75 次调用即放行
   （`orchestrator.py:61-65,147-159`），PaperBench 两跑 19-32 分钟收工（预算 3600s）；
   SAB 以 40 次调用上限 353s 收工（超时额 1200s）。
4. **无 sanity gate**：stochastic-interpolants 自测 4/9 报错（含定义性性质 I_0≠x0）仍交付；
   pinn 假训练（adam 与 adam+lbfgs 损失**逐位相同**）agent 不读不报；
   自进化报告在零执行下**虚构** Bader 电荷/MD 数据（`engine.py:5016-5022` 无数据硬门）——这比空转更危险。
5. **给了答案也落实不了**：PaperBench prompt 明文给的 4 条答案，最终代码 4 条全错
   （`simformer.py:29,37,64`、`train.py:59`），跨三代运行未修——长上下文指令落实能力的硬证据。

## 三、工作流与 harness 配置压制（最大可修复杠杆）

1. **模式切换从未发生**：四个适配器均无 `set_mode` 调用，exec_mode 恒 `tool_call`、user_mode 恒 `chat`；
   PaperBench 残留 1321 条 checkpoint 中 `[PHASE:` 标记 0 命中。**「切换丢上下文」的直觉不成立。**
   真正的模式层问题是「空转与断路」：
   - phase-aware 预算通道 100% 死代码（`bench/orchestrator.py:171-173` 要求 research mode）——
     设计中的 EXECUTION 300-calls 相位预算从未生效，SAB/MLE 反被均一低预算帽提前闷死；
   - 反思驱动的模式切换因类型不匹配（`discover/construct` vs `chat/research/plan`）永远
     ValueError 被静默吞掉（`reflection.py:319`）——自适应纠偏闭环自始死锁。
2. **模型静默降级**：全部 benchmark 实跑 deepseek-chat（非推理、temp 0.7），
   toml 里的 deepseek-reasoner 被 `from_env` 静默忽略——对标官方 Claude/GPT-4o 级差两个档。
3. **检索全灭**：physics bench 三道错题（含 5 个数量级、符号错误）全部伴随 web_search
   四级降级链全灭（ddgs 改名+bing 超时），agent 无「搜索不可用」信号，耗死重试后裸答猜值。
4. **数学工具被结构性摘除**：四个适配器的 tool_filter 把 symbolic_math/bourbaki/lean/unit/validate
   全部过滤（`sab_huginn.py:53-62` 等）——30 份 RCB 轨迹 grep 数学工具 **0 命中**。
5. **Windows 层**：长任务 30 轮 10 败，40% 为 WinError 2 穿透 sandbox→adapter→HTTP 三层整轮作废；
   PaperBench M1 第 16/150 次调用死于 checkpoint 悬空 tool_calls（400 不重试不修复）——
   这是**预算扩容的前置修复项**。

## 四、你的两个核心目标：自进化与数学直觉，目前都是断线的

### 自进化：「空转」而非「进化」【06·11】
- 18/18 轨迹只有 perceive+report 两阶段、0 工具调用（1.6-11s 玩具 loop）：
  `_perceive()` 以 git 文件变更为触发源，干净 workspace 下 hypothesize→learn **从未执行**
  （`engine.py:1353-1362, 2304`）；
- 进化知识是噪声：25 条规则全是 `read_file not found` 模板匹配产物，无 LLM 蒸馏，usage_count 全 0；
- **写入/读取路径永不相交**：引擎写 `~/.huginn/logs/`，context_builder 读 `$HUGINN_CACHE_DIR`（:400）；
- 四个 benchmark 适配器对 evolution/reflect **零引用**——每次运行都是无记忆一锤子买卖；
- 唯一有效的「进化」是人工 prompt 修补，且部分有害（spaceship 0.6833→0.638 回退、rubric 答案泄漏）。

### 数学直觉：器官齐全、神经未接、反射弧语义为空【13】
- 子能力评分（实现/接入）：符号推导 6/2、量纲 5/1、守恒 3/2、形式化证明 2/1、数值-符号互验 3/1；
- 已接线的 Lean 守恒检查是**重言式**（硬编码恒等演化，`bourbaki_tool.py:116-132`），
  定理验证子串匹配、sorry 可通过——「验证通过」当前不携带信息量；
- repro 两题数量级错误（χ=1.0 照抄题面常数、θD=6.6K 丢指数）发生在**工具全量可用**的内部基准上——
  断点在行为层：缺「数值答案必须经工具计算」的强制 gate，最短闭环是 ~50 行确定性 gate，不是接 Lean。

## 六、修复路线图（按 ROI 排序，跨 13 份报告共识）

### 第一梯队：小时级改动，立见分
1. 删 `sab_huginn.py:311` 的 `code[:8000]`（+30-40 分测量回收）；
2. 预算错误消息加重置语义 + MLE 60→150 calls、SAB 40→对齐超时额；
3. 完成判据换成 rubric 覆盖清单 + sanity gate（自测全绿/loss 真降/无占位数值才准停）——
   对标 ResearchClaw 的 remediation task 与 AutoResearchClaw 的 AST 门控；
4. 报告-产物一致性核对器（拦截 "Expected"/虚构数据）；
5. 修 `from_env` 让 reasoner 模型生效；MLE medal 死代码修复；
6. may-2022 提交 `test_probs` 而非硬标签（一行，+0.04 AUC）。

### 第二梯队：天级，修复执行链
7. Rust sandbox 静默崩溃纳入回退判定；熔断器自锁恢复路径；
8. checkpoint 悬空 tool_calls 修复（**预算扩容前置项**）；
9. tool_filter 恢复数学工具 + repro/科学任务加「数值必须经工具计算」确定性 gate；
10. web 检索链修复 + 「搜索不可用」显性信号；
11. 评测卫生：judge 换非同源模型、structai 依赖声明、泄漏修复（审计 16）——
    **在修复进化回路之前必须先修评测信号，否则将制度化 reward hacking**。

### 第三梯队：周级，接通你的两个目标
12. autoloop objective-driven 化（perceive 从文件变更驱动改为任务驱动）；
13. 进化回路三段修复：载体（真实任务执行）、写入（LLM 蒸馏+分数输入）、
    读取（路径统一）+ benchmark 失败→分析→更新→重试回路（对标 EvoScientist EMA 写入）；
14. 模式层接线或拆除：phase 预算通道接通（或显式移除死代码）、反思切换类型对齐；
15. 数学链深水区：量纲/守恒接真语义（替换重言式）、数值-符号互验。

> 原则提醒（对标 EvoScientist 的结论）：自进化的前提是「可执行性硬门」与「诚实的评测信号」。
> 当前两者都缺——先把执行闭环和评分管线修成可信，再谈进化，否则进化出来的是 reward hacking。
