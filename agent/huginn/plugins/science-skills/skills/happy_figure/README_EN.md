Turn papers, captions, proposals, research plans, and reference figures into copy-ready, reviewable, controllable prompts for AI scientific illustration.

`happy-figure-skill` is the Agent Skill companion to [Happy Figure](https://github.com/datawhalechina/happy-figure). It does not generate images directly. Instead, it helps Claude Code / Codex-style agents read research content first, then produce structured figure prompts adapted to the **research domain x figure type x image model** you want to use, including Nano Banana Pro, Nano Banana 2, Qwen Image 2 / 2 Pro, and GPT Image 2.

In one sentence: this is not another generic "make a scientific figure" prompt. It is a **prompt router and figure-structure compiler** for AI scientific illustration.

## 🖼️ Showcase

We selected 5 representative scientific figure tasks and ran them through Happy Figure Skill. The images below were generated by sending the resulting prompts to 5 different image models.

This showcase is meant to prove two things at once: Happy Figure Skill can cover multiple domains and figure types, and the same prompt will still diverge across models because each model has its own default visual language, text behavior, and structural bias.

- It can turn CS/ML, materials chemistry, biomedicine, earth science, and other research content into executable figure prompts.
- It helps you choose a better generation route based on domain, figure type, and model behavior.

> 🔎 Full prompts and the 5 x 5 mapping table live in [showcase assets](https://github.com/BAIKEMARK/happy-figure-showcase-assets/tree/main/prompts). This repository keeps only the Skill itself.

> 💡 Tips: models from the same family usually preserve a similar default visual style. Qwen 2 models have a distinctive scientific illustration texture and are affordable enough for multiple sampling rounds, but complex figures may break apart and text errors rise quickly as labels increase. Nano models tend to look like flat AI scientific illustrations and may render structural prompt words such as `ZONE` into the image. GPT Image 2 is generally stronger for text and structure, with a cleaner vector-like look that is visually different from Nano and Qwen.

> ⚠️ Warning: these images are model outputs, not submission-ready figures. Before publication, authors should manually verify scientific structure, labels, arrow semantics, data boundaries, and journal requirements, then redraw, vectorize, or re-typeset when needed.

## 🧭 Why It Is Different

Most generic prompts stop at "generate a scientific schematic." Happy Figure Skill first makes three routing decisions:

| Layer | What it checks | What it controls |
| --- | --- | --- |
| Research domain | CS/ML, materials chemistry, biomedicine, or unknown/interdisciplinary domains | Scientific objects, publication context, and domain-specific visual language |
| Figure type | Workflow, apparatus, mechanism, multi-panel comparison, graphical abstract, overview figure, cover art | Layout, regions, arrows, panels, visible text, and failure boundaries |
| Image model | Nano Banana Pro, Nano Banana 2, Qwen Image 2 / Pro, GPT Image 2 | Label density, structure constraints, visual goal, and text strategy |

For example:

```text
Use $happy-figure-skill. Read this methods section and generate a materials chemistry mechanism figure prompt.
Target model: Nano Banana Pro. Keep only short English labels in the figure.
```

It will not turn a materials mechanism into a generic flowchart, nor will it force a CS model architecture into a Nature/Cell biomedical style. Domain masters handle scientific meaning, figure-type masters handle the communication task, and model cards handle the final generation adaptation.

## 🎛️ Choosing a Model

There is no absolute "best" model. The better question is what kind of figure you need.

| You care most about | Try first | Why |
| --- | --- | --- |
| High-impact graphical abstract, strong first impression | Qwen Image 2 Pro, Nano Banana Pro | More likely to produce layered, polished scientific visuals |
| Complex structure and final candidate stability | Nano Banana Pro | Better for multi-region, hierarchical, arrow-heavy figures |
| Fast iteration and composition exploration | Nano Banana 2, Qwen Image 2 | Good for generating several candidates before choosing a direction |
| Chinese or bilingual style drafts | Qwen Image 2, Qwen Image 2 Pro | Good Chinese semantic understanding, but keep visible text sparse |
| Clear module relationships, more labels, redraw baseline | GPT Image 2 | More controllable for system diagrams, structured baselines, and text-rich figures |

You can also use the showcase above as a visual reference for model selection. If you are unsure, ask the Skill for a model-neutral structured prompt first; after the first batch of images, iterate toward "more polished", "clearer structure", "less text", "more like a paper figure", or "more like a graphical abstract."

## 🧩 Supported Figure Types

| Figure type | Best for | What the Skill controls |
| --- | --- | --- |
| Technical roadmap / research workflow | Methods, research plans, project routes, data-processing pipelines | Step count, reading direction, branches/feedback, inputs and outputs |
| Experimental system / apparatus schematic | Experimental platforms, instruments, acquisition systems, sample transfer | Real components, connections, material/signal/data flow |
| Mechanism explanation | Reaction paths, regulatory pathways, material evolution, disease mechanisms | Entity forms, causal chains, activation/inhibition/conversion arrows |
| Multi-panel comparison | Comparisons, groups, ablations, conditions | Panel consistency, scale, group names, data boundaries |
| Graphical abstract / main figure | Core contribution, project homepage visual, submission abstract figure | Research problem, core method, key process, visual hierarchy |
| Proposal / defense / presentation overview | Topic structure, research modules, expected outcomes, group meeting summaries | Overview hierarchy, module relationships, presentation readability |
| Journal cover / cover art | Cover art, teaser, promotional scientific visual | Visual metaphor, strong central object, no fake journal logo/issue/barcode |

## 🌐 Supported Domains

| Domain | Visual language | Best for |
| --- | --- | --- |
| Computer science / machine learning | ACM/IEEE/NeurIPS/ICML/CVPR/ACL-style paper figures; modules, data flow, tokens, KGs, pipelines, system boundaries | Model architectures, LLM pipelines, knowledge graph systems, information retrieval, databases, document processing |
| Materials and chemistry | Nature Materials / Advanced Materials style; lattices, interfaces, ion migration, solvation shells, reaction paths | Electrochemistry, catalysis, crystal/interface mechanisms, multiscale material structures, reaction pathway figures |
| Biology and medicine | BioRender / Cell / Nature style; cells, proteins, pathways, tissue microenvironments, drug mechanisms | Signaling pathways, disease mechanisms, clinical flow, drug action, cellular microenvironments |
| Unknown or interdisciplinary fields | Adaptive domain masters generated from source material and domain semantics | Earth science, environment, energy systems, agriculture, psychology, public health, policy modeling |
| Reference-image style transfer | Transfers only reusable layout, palette, line, typography, and hierarchy; never copies scientific content | "Draw it in the style of this reference" while preserving your paper's own scientific structure |

## ⚡ Quick Start

```bash
npx skills add BAIKEMARK/happy-figure-skill
```

After installation, ask Claude Code / Codex:

```text
Use $happy-figure-skill. Read the following paper abstract and generate a graphical abstract prompt for Nano Banana Pro.
```

You can also ask an agent with shell access to install it for you:


## 🧪 Example Prompts

Specify a figure type:

```text
Use $happy-figure-skill. Based on this methods section, generate a technical roadmap figure prompt.
Keep only short English labels inside the figure. Do not generate long explanatory text.
```

Specify domain and publication context:

```text
Use $happy-figure-skill. Convert this materials chemistry mechanism description into a Nature Materials-style mechanism schematic prompt.
Target model: Qwen Image 2 Pro.
```

Specify model behavior:

```text
Use $happy-figure-skill. Generate a model architecture figure prompt from this CS/ML paper.
Target model: GPT Image 2. Prioritize clear module relationships and readable labels.
```

Only output the final prompt:

```text
Use $happy-figure-skill. Only output the final image-generation prompt. Do not explain the process.
```

Use a file:

```text
Use $happy-figure-skill. Read ./paper.pdf, extract the abstract, methods, captions, and conclusion, then generate a graphical abstract prompt.
```

## 🔁 Workflow

1. Read research content: abstracts, methods, captions, research plans, and files such as `.pdf`, `.docx`, `.tex`, `.md`, and `.txt`.
2. Build a Figure Brief: research topic, domain, figure type, target model, visible-text language, and scientific boundaries.
3. Select masters: domain masters define scientific objects and domain visual language; figure-type masters define layout and communication structure.
4. Adapt to the model: adjust label density, structure constraints, text strategy, and style intensity.
5. Output the final prompt: visible-text whitelist, forbidden content, arrow semantics, region structure, and rendering style.
6. Add a short review reminder: check scientific structure, labels, arrows, data boundaries, and journal requirements.

## 🚧 What It Does Not Do

This Skill focuses on generating figure prompts from research documents. It does not pretend to solve every scientific-figure problem.

| Not suitable for | Why |
| --- | --- |
| Real data plots, statistical curves, significance markers | Image models must not invent data, axes, p-values, or experimental results |
| Microscopy images, experiment photos, real equipment photos | These are evidence images and should not be fabricated |
| Strict engineering drawings or exact geometric dimensions | Generated images can guide composition, but not replace precise engineering drawings |
| Direct submission of raw PNG outputs | Submission figures still need human review, redrawing, vectorization, or re-typesetting |
| Watermark removal or fake journal-cover elements | It should not generate journal names, logos, issue numbers, barcodes, or misleading publication artifacts |

## 📚 Relationship to Happy Figure

Happy Figure teaches the methodology: how to understand scientific illustration, design prompts, control image generation, post-process AI-generated figures, and handle academic compliance. Once you understand Happy Figure, you can build a controllable, reviewable, publication-oriented AI scientific-figure workflow.

`happy-figure-skill` turns the prompt-writing part of that methodology into an Agent Skill: read the paper, choose the figure type, select domain visual language, adapt to model behavior, and output a copy-ready figure prompt.

The tutorial helps you understand why to draw this way. The Skill helps you start drawing faster.
