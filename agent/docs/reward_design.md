# 形式化-物理双轨奖励设计

> 状态：理论设计稿，未实现。用于讨论 Agent RL 训练中"数学结构化"信号如何安全地辅助物理/化学任务。
> 关联讨论：数学结构化作为辅助奖励 vs 世界模型表征学习（JEPA）的联系与张力。

## 1. 问题陈述

RL 训练 Agent 需要高频反馈。在材料/化学任务里，物理反馈周期长（一次 DFT/MD 分钟到小时级），梯度信号稀疏，导致：

- 编程类任务成了"训练场"（编译/测试信号密集），物理/化学成了"应用目标"（信号稀疏）
- 直接用物理结果做唯一奖励，采样效率极低，难收敛

朴素方案是用"数学结构化"信号替代——agent 能否把物理问题写成方程、推导是否自洽。这信号密集可验证，但有两个已知失败模式：

1. **数学自洽 ≠ 物理正确**：推导漂亮的模型可能选错哈密顿量/边界条件，物理全错
2. **奖励黑客**：agent 学会把任何东西包装成"数学结构"骗奖励，而非解决物理问题

本设计的目标：**用数学结构化的密集信号填补梯度空洞，同时用物理一致性锚定最终裁判，避免 agent 在数学空间过拟合。**

## 2. 核心思路：双轨 + 对齐桥梁

不二选一，建两条奖励轨道 + 一个桥梁：

```
┌─────────────────────────────────────────────────────────┐
│  形式化轨道 (PRM, 密集)    物理轨道 (ORM, 稀疏)         │
│  练表征能力                判对错                       │
│         └──── 对齐桥梁 (双向校准) ────┘                │
│         奖励"形式化预测能预测物理结果"                  │
└─────────────────────────────────────────────────────────┘
```

- **形式化轨道**：过程奖励，信号密集，职责是练"把物理翻译成可验证数学对象"的能力
- **物理轨道**：结果奖励，信号稀疏但权威，职责是判对错
- **对齐桥梁**：奖励形式化预测与物理结果的吻合度，同时惩罚两种脱节

桥梁是关键。没有它，两条轨道各跑各的，agent 会在数学空间过拟合。

## 3. 形式化轨道（PRM）

**信号源**（每个可独立验证、密集可得）：

| 信号 | 验证方式 | 工具 |
|------|----------|------|
| 方程是否建立 | 形式化语法检查 | `symbolic_math_tool` |
| 量纲是否一致 | 量纲分析 | `autodiff_tool` |
| 推导是否被证明器检查通过 | Lean4/Coq 验证 | `lean4` skill |
| 符号推导是否自洽 | 符号回归对比 | `symbolic_regression_tool` |

**reward 定义**：

```
R_formal = w1·1[方程建立] + w2·1[量纲一致] + w3·1[证明通过] + w4·sim(推导, 已知解)
```

**职责边界**：只奖励"形式化能力是否练到位"，**不**奖励"物理是否对"。明确这条边界是为了防止它越权替代物理裁判。

## 4. 物理轨道（ORM）

**信号源**（稀疏、权威）：

| 信号 | 验证方式 | 工具 |
|------|----------|------|
| 守恒律 | 能量/动量/粒子数守恒检查 | `validate_tool` |
| 极限行为 | 已知极限（零场、高温、长时）渐近 | `validate_tool` |
| Born 稳定性 | 弹性常数判据 | `mechanics` (BornStabilityChecker) |
| 与基准/实验偏差 | 数值对比 | simulation + 基准库 |

**reward 定义**：

```
R_phys = -α·||预测 - 基准||_huber - β·Σ 1[守恒律违反] - γ·1[稳定性失败]
```

注意用 Huber 而非 MSE，理由见 §8。

**职责边界**：只判对错，**不**负责练能力。能力由 PRM 练，对错由 ORM 判。

## 5. 对齐桥梁（关键组件）

**reward 定义**：

```
R_align = agreement( 形式化预测, 物理模拟结果 )
        - λ1·1[形式化漂亮 ∧ 物理错]   # 惩罚黑客
        - λ2·1[物理对 ∧ 形式化草率]   # 惩罚没建表征
```

**双向惩罚的设计意图**：

- 第一项鼓励形式化预测真的能预测物理——这是 agent 是否"理解"了物理的硬检验
- 第二项防止 agent 跳过形式化直接猜物理答案（比如背基准值）——强制它走"建模型→预测"的路径
- 没有第二项，agent 会退化为查表；没有第一项，agent 会退化为数学游戏

**信用分配难点（必须正视）**：形式化预测和物理结果吻合时，到底奖励"形式化建得好"还是"碰巧对齐"？这是本设计最大的不确定点。建议先用离线 trace 做 reward shaping 消融（见 §9），确认桥梁奖励确实把 agent 往物理正确方向推，再上线 RL。

## 6. 阶段化权重

不同训练阶段，三条轨道权重变化：

| 阶段 | R_formal | R_align | R_phys | 目标 |
|------|----------|---------|--------|------|
| 早期 | 高 | 低 | 低 | 练表征能力（curriculum） |
| 中期 | 中 | 升高 | 中 | 强制对齐 |
| 晚期 | 低（降为约束） | 中 | 高 | 对齐物理目标 |

**早期重形式化**：梯度空洞最严重，先用密集信号把"形式化能力"练熟。
**晚期形式化降为约束**：从奖励变成硬约束（形式化必须通过，但不额外奖励），主导权交给物理。这避免 agent 停在"数学漂亮"的局部最优。

## 7. 反事实层（最后加）

桥梁稳定后，加主动干预式反事实：

```
agent 主动改参数/边界条件 → 用形式化模型预测物理变化 → 用模拟验证
```

这对应从"被动预测误差"走向"主动反事实推理"的能动性跃迁。**不要一开始就做**——反事实信号比预测误差更稀疏，早期训不动。先靠 PRM + 桥梁把世界模型稳住，再加 agency 层。

## 8. 重尾 Loss 设计（直接可操作）

这是塔勒布"关注尾部"在 loss 层的具体实现，也是对"高斯假设"张力的工程化解：

- **PRM 层（表征空间）**：可用高斯残差假设。表征空间已去噪，残差近似高斯合理
- **ORM 层（物理结果）**：**必须重尾鲁棒**。用分位数损失 / Huber，**不用纯 MSE**

理由：相变、失效、极端事件是分布的尾部。纯 MSE 在高斯假设下会系统性低估这些事件的损失权重，agent 学会"在中心拟合得好、尾部完全错"。Huber 在残差大时退化为 L1，不被极端值支配；分位数损失可直接指定关注哪个尾部。

```
# 不要这样（高斯，抹平尾部）
L_orm = MSE(预测, 基准)

# 要这样（重尾鲁棒）
L_orm = Huber(预测, 基准, δ) 或 QuantileLoss(预测, 基准, τ=0.05/0.95)
```

## 9. 风险与消融建议

**主要风险**：

1. **桥梁信用分配**（最大风险）：见 §5。形式化-物理吻合的归因难
2. **奖励黑客**：形式化轨道权重过高时，agent 会包装数学骗奖励
3. **阶段切换震荡**：权重过渡太快，agent 能力跟不上，训练崩溃

**消融建议（上线 RL 前必做）**：

- 离线收集 agent 的执行 trace（形式化过程 + 物理结果）
- 对比四种 reward 配置：纯 PRM / 纯 ORM / PRM+ORM 无桥梁 / PRM+ORM+桥梁
- 验证桥梁配置下，agent 的物理正确率确实上升，而非只是形式化评分上升
- 只有桥梁配置在"物理正确率"指标上显著优于无桥梁，才上线 RL

## 10. 与项目工具的映射

本设计不是空中楼阁，项目里已有执行端支撑：

| 轨道 | 信号 | 项目工具 |
|------|------|----------|
| 形式化 PRM | 方程/量纲/推导 | `symbolic_math_tool`, `autodiff_tool`, `symbolic_regression_tool`, `lean4` skill |
| 物理 ORM | 守恒律/稳定性 | `validate_tool`（含 Born 判据）, `mechanics`, `scientific-simulation` skill |
| 对齐桥梁 | 形式化-物理吻合 | **待实现** —— 需要一个比对形式化预测与模拟结果的组件 |
| 反事实 | 干预-预测-验证 | **待实现** —— 需要参数扫描 + 模拟批跑 |

桥梁和反事实层是后续工程化的重点，也是本设计能否落地的关键。

## 11. 开放问题

1. 形式化预测与物理结果的"吻合度"如何定义？不同物理量（能量 vs 谱 vs 结构）度量不同
2. 阶段切换的触发条件：按步数、按 R_formal 收敛、按 R_align 稳定？需实验定
3. 多物理场耦合时，桥梁奖励如何分解到各子模型？
4. 与 JEPA 表征学习的关系：形式化轨道是否本质上在帮 agent 建"几何表征空间"？若是，能否用 JEPA 的预测误差替代部分 PRM 信号？这是后续理论方向

---

**结论**：数学结构化作为辅助奖励是合理的，但必须：(1) 作为 PRM 而非 ORM 替代；(2) 用对齐桥梁锚定物理裁判；(3) ORM 层用重尾 loss 保护尾部；(4) 上线 RL 前做消融验证桥梁有效性。脱离这四条，会训出一个"数学很好但不懂物理"的 agent。
